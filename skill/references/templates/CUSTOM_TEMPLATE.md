# Custom Template

Юзер сам предлагает что хочет видеть в базе знаний. Skill строит под него.

## Примеры кастомных запросов которые встречались

### "Карта всех смертей в шоу с причинами и поэтикой"

Файлы:
- deaths_ledger.yaml - каждая смерть: кто умер, как, кто убил/причина, эмоциональный вес, тематическое значение
- deaths_by_season.md - таблица по сезонам
- weapon_methods.md - чем убивали и как
- emotional_impact_ranking.md - какие смерти ударили зрителей больше всего

### "Где найти спокойствие если ты главный герой?"

Файлы:
- safe_havens.md - локации где никого не убивали
- moments_of_peace.md - сцены покоя в эпизодах
- emotional_breaks.md - где главный герой отдыхал

### "Музыкальный гид по сериалу"

Файлы:
- soundtrack_by_episode.md
- recurring_motifs.md - повторяющиеся muzyk-themes
- character_themes.md - какая музыка ассоциирована с каждым героем
- standout_moments.md - сцены где музыка была ключевой

### "Bran/Daenerys/X parallel paths analysis"

Файлы:
- parallel_journey.md - сравнительная timeline двух персонажей
- shared_themes.md - общие мотивы их арок
- divergence_points.md - где их пути разошлись

### "Кто из персонажей выжил бы в реальности?"

Файлы:
- survival_analysis.md - per-character оценка реалистичности survival
- skills_inventory.md - реальные навыки персонажей
- death_probability.md - какова была реальная вероятность смерти каждого героя

## Алгоритм когда юзер выбирает F (Custom)

1. Спроси юзера развёрнуто что он хочет: "Опиши идею в 2-3 предложениях. Что хочешь видеть в базе? Какие связи между чем?"
2. Сформулируй спецификацию (5-10 файлов под этот запрос)
3. Покажи юзеру: "Я понял так: построю файлы X, Y, Z. Подтверди или поправь."
4. После подтверждения - построй базу

## Принципы кастомных баз

- Sticking to markdown + frontmatter + [[wiki-links]] - всегда
- Каждый файл имеет своё назначение
- Один index.md с tour по всем файлам базы
- Если запрос требует анализа которого нет в wiki (например музыка, которой нет в fandom) - искать в reddit / web search / посоветовать юзеру дополнить
